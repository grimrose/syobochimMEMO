用語MEMO¶

単語集
IHS IBM HTTP Server
WAS (WebSphere Application Server)Webシステムの構築や運用をサポートするためのアプリケーション・サーバー・ソフトウェアです。
Java EE Java Platform, Enterprise Editionの略で、Javaによる、サーバーで稼動する業務用システム（エンタープライズ・システム）の構築のための標準仕様です。Javaの基本機能を備えたJava SE(Java Platform, Standard Edition)を基盤にして、さらにアプリケーション・サーバーにおいて提供されるべき機能の仕様と、その機能をJavaプログラムから利用する方法（API）を定義しています。
コンポーネント アプリケーションを構成する部品であり、開発者が作成するJavaのプログラムです。それらプログラムを複数まとめて1つのコンポーネントとも呼ぶ場合もあります。
コンテナー アプリケーション・サーバー・ソフトウェアが提供する、コンポーネントの実行環境です。コンテナーは、コンポーネントの動作に必要な一般的な機能、例えば、Webブラウザーからの入力データに対する統一的なアクセス、データベース接続管理、セキュリティー管理などを提供します。
Controller リクエストの受付から、レスポンスを返すまでの全体の制御を行います。クライアントがブラウザーで入力した値を受け付け、適切なビジネスロジックを実行するEJBなどに処理を依頼し、クライアントへの返答ページの作成をJSPに依頼します。この役割を担当するのが、サーブレットです。
Model データベースなどのバックエンドシステムへアクセスをしたり、値の計算などのビジネスロジックを実行したりします。この役割を担当するのは、EJBです。また、EJBの代わりに、ビジネスロジックを実行する単なるJavaのクラスが、この役割を担うこともあります。
View クライアントへ返答するWebページの作成を行います。JSPが担当します。
EJB Enterprise JavaBeansの略です。EJBは純粋にビジネスロジックを記述した、サーバー・サイドのJavaの部品です。データベースなどの既存のシステムと連携したビジネスロジックなどを実行します
セル セルは、WAS の1つ以上のノードを論理的なグループにまとめたものです。WAS ND構成で使用されるものであり、WAS Baseでは通常は意識しません。
ノード ノードは、WASが管轄する論理的なマシンです。多くの場合、ネットワーク上で識別可能な IP ホスト・アドレスを持つ物理的なマシンに一致します。NDの複数台構成では、あるノードをあるセル配下に構成することを、「統合する」といいます。各ノードで実行するアプリケーション・サーバーの管理は、DMがノード・エージェントと通信することにより行います。
Webサーバー Webサーバーは、HTTPリクエストを受け付けるhttpdプロセスです。 WASにはApacheをベースにしたIBM HTTP Server(IHS)が同梱されています。IHS以外のWebサーバーの使用も可能です。
デプロイメント・マネージャー（DM） DMは、セル内のすべての構成要素の集中管理を行います。DMは、それ自体が専用のアプリケーション・サーバー上で実行されていて、Webベースの管理インターフェース（管理コンソール）などの管理機能を提供しています。
ノード・エージェント ノード・エージェントはDMからの管理要求をアプリケーション・サーバーに伝達するJavaプロセスで、セルに統合されているすべてのノードで実行されます。また、ファイル転送サービス、構成の同期、およびパフォーマンス・モニターなどの管理機能を提供します。
Edge Components Edge Componentsは、様々なサービスやシステムに対するネットワーク・トラフィックの負荷分散機能（Load Balancerコンポーネント）を提供します。
セッション・アフィニティー セッションを使用したアプリケーションに対して、同一セッションIDを持つクライアントからの要求を毎回同じクラスター・メンバーに渡す方式です。
セッション・パーシスタンス セッション情報を外部のデータストア（DBまたは他のアプリケーション・サーバー）に格納することによって、複数のアプリケーション・サーバーでセッション情報を共有（セッションのクラスター化）します。
EIS(Enterprise Information Service) データベースやメッセージング製品など、アプリケーション・サーバーの外部に存在するシステム
JDBC(Java Data Base Connectivity) リレーショナル・データベースへ接続を行うためのAPI
JDBCドライバー JDBCのAPIと各データベース製品を結びつけるクラス群がJDBCドライバーです。JDBCドライバーは、JarファイルやZipファイルといった形で、各データベース・ベンダーやサードパーティーによって提供されています。
JNDI Java Naming and Directory Interfaceの略で、ディレクトリー・サービスが提供するデータやオブジェクトを名前で検索し、参照するためにJava EEで定められた仕様です。
接続プール アプリケーションで作成される接続オブジェクトを再利用する仕組みです。接続プールを利用しない場合、生成された接続オブジェクトはデータベースの処理を終えると削除されますが、接続プールを利用した場合は、データベースの処理を終えた接続オブジェクトは接続プールに一定期間保管されます。一定期間内であれば、次に接続オブジェクトが必要となった時にプール内にある接続オブジェクトを再利用することができます。接続プールはデータソースを設定する時の設定項目の一つであるため、アプリケーション・サーバー上でデータソースを設定した際に同時に設定可能です。
JMS（Java Messaging Service） JMSはJava EE標準のプログラミング・インターフェースです。
PTP Point To Point。メールに例えられる1対1の通信形態です。メッセージのやり取りはキューを介して行われます。送信者(Sender)は、キューに対してメッセージを送信します。受信者(Receiver)は、必要なときにキューからメッセージを受け取り、処理を行います。
Pub/Sub Publish/Subscribe。メーリング・リストに例えられる1対多の通信形態です。発行者(Publisher)は、トピックに対してメッセージを送信します。全ての購読者(Subscriber)は、前もってトピックに購読依頼を出しておくことで、メッセージを受け取り、処理を行います。
JMSクライアント JMSインターフェースを利用してメッセージング・サービスを利用するアプリケーションのことです。
JMSプロバイダー JMSのインターフェースを提供するJava EE側のコンポーネントを指していて、アプリケーションがどのメッセージング・サービス上の宛先に対して送受信を行うかという接続機能を提供します。具体的には後述のJMS接続ファクトリーを提供し、JMS宛先に対する接続を作成します。(例：WebSphere MQ Messaging Provider(WMQ用))
JMS接続ファクトリー JMSプロバイダーの接続を管理しており、接続するメッセージング・サービスのキュー・マネージャー名やホスト名などの接続情報を保持しています。
MDB(Message-Driven Bean) JMS APIを利用したメッセージ駆動型処理を行うEJBです。Web環境において、業務ロジックを非同期に処理する際によく利用されます。MDBはメッセージを受信した時にEJBコンテナーによって呼び出され、メッセージをEJBアプリケーションで処理することができます。また、並列にメッセージ受信およびビジネスロジックの実行が可能です。
プロトコル プロトコルは、表示する情報の種類を特定するものです。HTTP(Hyper Text Transfer Protocol)は、Web上のコンテンツにアクセスするためのプロトコルです。更に、安全に情報をやりとりするためのデータ暗号化機能が付加されているのがHTTPS(Hyper Text Transfer Protocol over Secure Socket Layer)です。
ポート番号 ポート番号は、接続先サーバーが提供しているサービス固有の番号です。通常、HTTP通信先となるWebサーバーのポート番号は80であり、well-known portとして決められているため、URL入力時に省略することができます。同様に、HTTPS通信の際のポート番号は443であり、こちらもwell-known portとして決められているため省略することができます
共通鍵暗号化方式 データの送り手と受け手が事前に同じ鍵を保有し、暗号化と復号をその同じ鍵で行う方式
Request for Comments（略称：RFC） IETF（Internet Engineering Task Force）による技術仕様の保存、公開形式である。内容には特に制限はないが、プロトコルやファイルフォーマットが主に扱われる。
スクリプトレット JSP内に書かれたJavaプログラム。<% %>で囲んで記述する。
サーブレット(Servlet) Javaで作られたHTMLなどのWebコンテンツを生成するためのプログラム。
J2EE バージョン5.0までの旧称で、以降はJava EEと呼ばれる。
JavaBeans Activation Framework(JAF) JavaMailなど他のパッケージによって使用される活性化フレームワークのAPI。任意データの方の判断、そのデータに対するアクセス手段のカプセル化、そのデータに対して提供される操作方法の取得、これらの総相談させてください。これらの操作を実行するための適切なBeanのインスタンス化、といった目的で使用する。
Java Transaction API(JTA) ドランザ区所ん管理を行うための一連のAPI
JMS MOMと通信し、システム間でのポイントツーポイントおよびパブリッシュ/サブスクライブ(発行と購読)のメッセージ交換を可能にするAPI
Java Naming and Directory Interface(JNDI) さまざまな種類のネーミング及びディレクトリサービスにアクセスするための統一インターフェースJNDIを使用して、J2EE環境のビジネスコンポーネントやその他のサービス指向オブジェクトの、登録及び検索などを行うｋとが出来ます。JNDIには、Lightweight Directory Access Protocol(LDAP)、CORBA Object Service(COS)ネーミングサービス、およびJava RMIレジストリのサポートが含まれている。
Intercepting Filter リクエストの前処理や後処理を行う
Front Controller リクエスト処理を集中して管理するためのコントローラを提供する
View Helper 表示用の整形に関係のないロジックをヘルパーコンポーネントにカプセル化する
Composite View 原始的名サブコンポーネントから構成されたビューを生成する
Service to Worker ディスパッチャコンポーネントを使ってFront ControllerパターンとView Helperパターンとを結合する
Dispatcher View ディスパッチャコンポーネントを使ってFront ControllerパターンとView Helperパターンと結合し、より多くの仕事をビューに処理させる
Business Delegate プレゼンテーション層とビジネス層を切り離し、サービスに対するファサードやプロキシの役割を持つインターフェースを提供する
Value Object ネットワークのトラフィックを減らして、層と層の間のデータ交換を行う
Session Facade ビジネスオブジェクトの複雑さを隠蔽するために、ワークフローの処理を一箇所にまとめる
Composite Entity 依存関係にある複数の永続オブジェクトを１つのエンティティBeanにまとめて、粗粒度のエンティティBeanを設計するという最良の方法
Value Object Assembler 複数のデータソースから複合バリューオブジェクトを組み立てる
Value List Handler 問い合わせの実行と結果のキャッシュ、および結果の処理方式を管理する
Service Locator 複雑なビジネスサービスの検索や生成の処理をカプセル化し、ビジネスサービスのファクトリを検索する
Data Access Object データソースを抽象化し、データに対する透過的なアクセスを提供する」
Service Activator EJBコンポーネントの非同期処理を行う
ディフェンシブ・パッケージング 上位クラスローダー配下に置いてある「クラスやリソース」の影響を受けないように、自アプリケーションに防衛策を施しておくパッケージング戦略のことです。アプリケーションのディフェンスが弱いと、容易に「侵入」を許してしまい、予期せぬ動作をしてしまうことがあります。
